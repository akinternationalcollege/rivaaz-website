1. PROJECT OVERVIEW
-------------------
Rivaaz is an event professional booking platform focused on Laxmangarh and surrounding cities (Sikar, Salasar, Fatehpur, etc.). 
The platform connects clients with verified professionals (waiters, chefs, dancers, security) for events like weddings and parties.
It is built as a highly premium, single-page web application (SPA) with no heavy backend—relying entirely on local browser storage for admin logs and the WhatsApp Business API (via direct URL redirects) for lead generation.

2. DESIGN SYSTEM & VIBE
-----------------------
- Theme: Ultra-Premium "OLED" Dark Mode.
- Colors: Deep Blacks (--ink:#050505), Glassy Cards (--ink-card:#1A1A1A), and Glowing Gold (--gold:#D4AF37, --gold-bright:#F9E596).
- Typography: 'Cormorant Garamond' for massive, cinematic headlines (clamp up to 7.5rem); 'Manrope' for clean UI text.
- Effects: Heavy use of Glassmorphism (backdrop-filter: blur(20px)) on the header and search bar.
- Animations: Deep shadow hovers that scale cards up, and a staggered "cascade" scroll reveal so elements glide up beautifully as the user scrolls.

5. CORE FUNCTIONALITIES
-----------------------
A. The Smart Calendar:
   - Native HTML5 date pickers used for booking dates.
   - 3-Day Rule: Users are blocked from booking events within the next 72 hours.
   - "Urgent" Override: A red checkbox allows users to bypass the 3-day rule for emergency Today/Tomorrow bookings.

B. The Vendor Funnel:
   - Clicking "Join as Professional" opens a massive, full-screen, highly styled overlay (z-index:2000) rather than a small popup.
   - Captures Name, Phone, City (Dropdown), Category, and Experience.

C. The WhatsApp Integration:
   - Forms do not send emails. When a user submits a Booking or a Vendor application, JavaScript builds a pre-formatted string.
   - It automatically executes `window.open('https://wa.me/916376213281?text=...')` to send the lead directly to the admin.
   - The WhatsApp number is dynamically pulled from the Admin settings (defaulting to 6376213281).

D. The Admin Dashboard (PIN: 9414):
   - Hidden at the bottom footer link "Admin Login".
   - It saves all settings and histories entirely via `localStorage` (wrapped in a `safeStorage` promise fallback).
   - TABS:
     1. Categories: Add/edit services and base rates.
     2. Professionals: Add/edit staff cards. Uses a "Service + Price + Call/Book" layout. The "Call" button uses `href="tel:+91..."`.
     3. Testimonials: Add/edit reviews.
     4. Settings: Update the Hero headline, subtext, and the master WhatsApp Number.
     5. Bookings: Permanent log of client event requests.
     6. Staff Inquiries: Permanent log of professionals applying for jobs.

7. PREVIOUS BUGS SOLVED (DO NOT REVERT THESE)
---------------------------------------------
- Bug: Date formatting script multiplied slashes (e.g., 10////1/02). 
  Fix: Removed the custom text-formatting script entirely and replaced it with native `<input type="date">`.
- Bug: Admin popups appearing behind the admin panel. 
  Fix: Ensure `.modal-overlay` has `z-index: 1000;` or `2000;` and `.admin-overlay` has `z-index: 600;`.
- Bug: PWABuilder rejecting the manifest. 
  Fix: Used reliable image hosting (dummyimage/github) and provided ALL optional fields (iarc_rating, handlers, etc.) in `manifest.json`.
- Bug: Vercel DNS failing. 
  Fix: Abandoned Vercel entirely. Switched to GitHub Pages A-Records.
